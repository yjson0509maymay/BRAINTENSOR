# Brain_Tensor 폴더 재구성 v2 - 실행순서 기준 재배치

작성일: 2026-07-03

## 1. 왜 다시 정리하나

1차 재구성(정리_작업_스크립트.ps1) 이후 `03_Model` 폴더 하나에 "모델 정의 / 학습 /
특징엔지니어링 호출 순서 / 분류 평가"가 섞여 있었고, `02_FeatureEngineering`이
`03_Model`보다 번호가 앞서지만 실제로는 `03_Model`의 학습이 끝난 뒤에 실행되는
단계라 번호와 실행순서가 어긋나 있었습니다. v2는 폴더 번호를 **실제 실행 순서**에
맞춰 재배치합니다.

## 2. 실행 순서와 폴더 번호의 대응

```
00_RawData          (원본 데이터, 손대지 않음)
   ↓
01_Preprocessing    (DICOM→NIfTI→BET→N4→FLIRT→정규화→리사이즈, 6단계)
   ↓
02_Model_Definition (3D-CNN·3D-ResNet 구조 "정의"만, 실행 로직 없음)
   ↓
03_Model_Training   (정의된 모델을 실제로 "학습" + 특징 추출까지)
   ↓
04_Feature_Engineering (03에서 나온 특징을 CCA로 융합 + WOA로 최적화)
   ↓
05_Model_Evaluation (최적화된 특징으로 SVM/KNN/GB/RF 추론 + k-fold 평가)
   ↓
06_Result           (평가 결과/성능표, 현재 비어있음 - 향후 실행 결과 저장)
   ↓
07_Document         (논문, 비교분석 문서 - 파이프라인 자체는 아니지만 참고자료)
   ↓
08_Visualization    (QC 시각자료 - 마찬가지로 참고자료)

지난파일              (아카이브, 번호 체계 밖)
```

`02_Model_Definition`과 `03_Model_Training`을 분리한 이유: "정의"(구조 코드, 몇 번을
재사용해도 바뀌지 않음)와 "실행"(학습 파라미터·로그·체크포인트처럼 실행할 때마다
달라짐)을 분리해두면, 나중에 모델 구조는 그대로 두고 학습만 다시 돌리거나 반대로
구조만 실험적으로 바꿔볼 때 어느 파일을 건드려야 하는지 명확해집니다.

"모델 추론(Inference)"을 "05_Model_Evaluation"에 합친 이유: 이 프로젝트는 k-fold
교차검증으로 평가하므로 각 fold마다 분류기를 학습하고(fit) 곧바로 그 fold의 테스트
데이터에 추론(predict)한 뒤 지표를 계산하는 것이 한 묶음으로 실행됩니다. 추론과
평가를 별도 폴더로 쪼개면 코드 실행 흐름과 폴더 구조가 오히려 더 어긋나서, 두 개념을
`ml_classifiers_kfold_eval.py` 하나에 유지했습니다.

## 3. 폴더별 상세 역할 및 파일 목록

### 00_RawData
- README_원본데이터안내.txt — 원본은 `RAW DATA` 폴더에 그대로 있다는 안내만 포함

### 01_Preprocessing (변경 없음)
- 전처리01_원본변환 ~ 전처리06_리사이즈_최종 (6단계 폴더)
- 로그_QC/, 스크립트/, data.csv, data_final_303.csv, README.md, PPMI_작업리스트.txt
- **정리됨**: `전처리06_리사이즈_최종`이라는 이름의 빈 폴더와 실제 데이터(304개)가 든
  `전처리06_리사이즈_최종_v2`가 공존했던 문제를 v2 스크립트가 정리합니다
  (빈 폴더는 지난파일로, `_v2`는 이름에서 v2를 떼어 진짜 최종 폴더명으로 변경)

### 02_Model_Definition — "무엇을 만들지" 정의
- `models.py` — CNN3D(3D-CNN), ResNet3D(3D-ResNet) 클래스 정의만 포함. 학습 로직 없음

### 03_Model_Training — "어떻게 학습시킬지" 실행
- `dataset.py` — PPMIT2Dataset, holdout/k-fold 분할 함수
- `train.py` — 전체 파이프라인 오케스트레이션(학습→특징추출→융합→최적화→평가 호출)
- `smoke_test.py` — CPU 환경 코드 검증용 경량 테스트
- `DEVIATIONS.md` — 논문과 다르게 결정한 모든 사항의 근거 기록
- `모델링_작업리스트.txt` — 진행 체크리스트
- `checkpoints/` — (신규, 현재 비어있음) 향후 학습된 .pth 저장 위치
- `training_logs/` — (신규, 현재 비어있음) 향후 epoch별 로그/리포트 저장 위치

### 04_Feature_Engineering — "특징을 어떻게 다듬을지"
- `cca_feature_fusion.py` (구 fusion.py) — CCA로 3D-CNN·3D-ResNet 특징 융합
- `woa_feature_selection.py` (구 feature_optimization.py) — WOA로 최적 특징 서브셋 선택

### 05_Model_Evaluation — "어떤 분류기로 얼마나 잘 맞는지"
- `ml_classifiers_kfold_eval.py` (구 classifiers.py) — SVM/KNN/GB/RF 학습+추론+k-fold 평가

### 06_Result (신규, 현재 비어있음)
- 향후 실제 학습/평가를 실행한 뒤 나오는 정량적 결과(성능표, 혼동행렬, JSON 등) 저장

### 07_Document (기존 05_Document, 번호만 변경)
- 전처리비교_논문_vs_실제.xlsx, 파이프라인비교_논문_vs_실제.xlsx
- 모델_아키텍처_분석.md/.pptx/.docx, 전처리_모델링_재현성_검증_보고서.docx
- 주논문_nature.pdf, 참고문헌 PDF 4종

### 08_Visualization (기존 06_Visualization, 번호만 변경)
- qc_*.png, qc_contact_sheets/, qc_images/, 303_rerun_wsl_visualization/

### 지난파일 (아카이브, 번호 체계 밖 - 이번에 추가되는 항목)
- `03_Model_구조체(빈폴더)`, `02_FeatureEngineering_구조체(빈폴더)`,
  `04_Result_구조체(빈폴더)`, `05_Document_구조체(빈폴더)`, `06_Visualization_구조체(빈폴더)`
  — 내용물을 다 옮기고 남은 이전 번호 체계의 빈 껍데기 폴더들
- `모델링_루트_구조체(빈폴더)`, `전처리_루트_구조체(빈폴더)` — 1차 재구성 때 완전히
  비워지지 않고 남아있던 루트의 옛 폴더 잔재
- `모델링_이전버전/특징벡터_npy/val_sample_ids.txt` — 1차 스크립트에서 누락되었던 파일
  (버그로 확인, 이번에 함께 정리)
- `모델링_이전버전/구버전_피처엔지니어링/fusion_optimization.py` — 새 코드로 대체된
  구버전 CCA+WOA 통합 스크립트
- `기타/정리작업_이력_v1/` — 1차 스크립트/로그/안내문 보관

## 4. 코드 상의 중요한 변경 사항 (폴더 분리에 따른 임포트 경로 수정)

`train.py`와 `smoke_test.py`는 각각 `models.py`(02), `dataset.py`(03, 자기 자신과
같은 폴더), `cca_feature_fusion.py`/`woa_feature_selection.py`(04),
`ml_classifiers_kfold_eval.py`(05)를 불러와야 하는데, 폴더를 분리하면서 같은
폴더에 있지 않은 모듈도 있게 되었습니다. 이를 위해 두 파일 모두 실행 시 자동으로
02/03/04/05 폴더를 파이썬 검색 경로에 추가하도록 이미 수정해 두었으니, 폴더 이동만
하면 별도 설정 없이 그대로 동작합니다(로컬 환경에서 재검증 완료).
