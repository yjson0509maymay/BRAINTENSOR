### 2-3. WOA (Whale Optimization Algorithm) — Figure 4 및 Eq.8-17 기준

| 단계(Stage) | 입력(Input) | 처리(Process) | 출력(Output) | 목적(Purpose) |
|---|---|---|---|---|
| Initialization | 전체 특징 집합 | Random initialization of feature subset for selection | 초기 특징 서브셋(고래 개체군) | 탐색 시작점 설정 |
| Population 생성 | 초기 특징 서브셋 | Assign each feature subset to whale position | 고래 위치로 매핑된 population | 각 후보해를 탐색 에이전트로 표현 |
| Fitness Function | 각 whale position | Calculate fitness function of each agent, Eq.(17): f(θ)=α·γ_R(D)+(1−α)·\|R\|/\|N\| (γ_R(D)=KNN 오류율, \|R\|=선택된 특징 수, \|N\|=전체 특징 수) | 각 agent의 적합도 값 | 정확도와 특징 수의 trade-off 평가 |
| 파라미터 초기화 | — | Initialize A⃗, D⃗, p, α | 초기화된 벡터/스칼라 값 | 탐색/개발 전환을 위한 계수 설정 |
| Exploitation(개발) — P≥0.5 | 현재 위치 | Update the position using spiral equation(Bubble-net attack, Eq.11-13) | 갱신된 위치 | 최적해 주변 나선형 탐색 |
| Exploration — P<0.5, \|A⃗\|≥1 | 현재 위치 | Select search position(무작위 agent 기반, Eq.14-16) | 갱신된 위치 | 전역 탐색(Exploration) |
| Exploitation(encircling prey) — P<0.5, \|A⃗\|<1 | 현재 위치, 최적 agent | Update best position and values A⃗, D⃗(Eq.8-10) | 갱신된 위치 및 계수 | 최적해 주변 포위(encircling prey) |
| Termination Condition | 반복 횟수 | If max iteration ≥ iteration 판정, 아니면 Calculate fitness function 단계로 복귀 | 반복 종료 여부 | 종료 조건 판정 |
| Best Solution(Output) | 최종 반복의 population | Best obtained position 산출 | 최적 특징 서브셋 | Feature selection 결과 산출 → Termination |

Table 7 파라미터(WOA): Constant(b)=1, Threshold(thres)=0.5, Population Size=100, Number of iterations=200, Search settings(Lower/upper bound)=0과 1, No. of independent runs=20.

## 3. 여러 모델이 함께 사용되는 경우 (전체 데이터 흐름, Figure 1 기준)

```
Input Data (303명 T2-weighted MRI, DICOM)
   ↓
Preprocessing (Brain Extraction → 이후 registration, normalization, augmentation; ref. 21,22 기술 방식 인용, 본문 p.9: DICOM→3D NIfTI 변환, skull stripping, field correction, 56×56×56 리사이즈)
   ↓  (동일 전처리 결과가 아래 두 모델에 병렬 입력됨)
   ├─────────────────────────────┬─────────────────────────────┐
   ↓                             ↓
[3D-CNN]                     [3D-ResNet]
 입력: 56×56×56×1 볼륨          입력: 논문에 기재되지 않음(3D-CNN과 동일 가정)
 처리: Conv3D×7+ReLU,          처리: Conv3D Block×2, Residual Block×5
       MaxPool×3, BN×3,              (각 3 Residual Unit), MaxPool×5
       Flatten, FC-1/FC-2
 출력: FC-1(1000) + FC-2(1000)  출력: FC-4(본문: 1000 / Fig.1: FV-4×2000)
   ↓                             ↓
Feature Concatenation(Eq.2)      │
 x̃3 = max(x1,x2), 중복 없음      │
 출력: FV-3(Fig.1 라벨: ×2000)   │
   └─────────────┬───────────────┘
                 ↓
      Feature Fusion (CCA, Eq.3-7)
       입력: FV-3, FV-4
       처리: 공분산행렬 S_AA,S_BB,S_AB → W_A,W_B → 고유값분해
       출력: 정준상관변수 Z1, Z2 (융합 특징, 본문 표기상 FC-3로도 지칭)
                 ↓
      Feature Optimization (WOA, Eq.8-17, Fig.4)
       입력: 융합 특징 벡터
       처리: Population 초기화 → Fitness(KNN 오류율 기반, Eq.17) →
             Exploration/Exploitation 반복 → 종료조건 판정
       출력: 최적 선택된 특징 서브셋(Best obtained position)
                 ↓
      Classification (SVM / KNN / GB / RF — Table 4, Table 6)
       처리: 각 ML 분류기로 지도학습 분류(GB가 대체로 최고 성능)
       출력: 예측 클래스
                 ↓
      Result & Evaluation (Control / PD / Prodromal, 3-class)
```

**⚠ 불일치 사항(논문 원문 내부 모순 — 임의로 해석/수정하지 않고 그대로 보고)**

1. **3D-ResNet 정확도(Table 8 vs 본문)**: Table 8에는 3D-ResNet=97.2%로 기재되어 있으나, 본문(p.10, Model-2 절)에는 "Gradient boosting-based ML algorithm achieved a maximum accuracy of 90%"라고 서술되어 있습니다. 97.2%는 Table 6의 Model 4(with optimization) 15-fold 결과 및 본문 p.14의 Model-4 최종 성능 서술과 정확히 일치합니다. 즉 Table 8의 "3D-ResNet" 행이 실제로는 Model-4(WOA 최적화 포함 최종 모델)의 수치와 동일한 값을 사용하고 있다는 정황이 있으나, 이는 관찰된 정황일 뿐 원문이 이를 명시적으로 설명하지 않으므로 임의로 재배치하지 않고 원문 그대로 기재합니다.
2. **Model-4 최종 성능의 Precision/Recall 순서(본문 p.14 vs Table 6)**: 본문 p.14는 "accuracy of 97.2%, precision of 90.6%, recall of 98.5%, and F1-score of 92.1%"라고 서술하나, Table 6의 Model 4 15-fold 행은 Accuracy=0.972, Recall=0.906, Precision=0.985, F1-Score=0.931입니다. Accuracy는 일치하지만 본문의 precision(90.6%)/recall(98.5%) 값이 Table 6의 recall(0.906)/precision(0.985) 값과 서로 자리가 바뀐 형태로 대응되며, F1-score도 92.1%(본문) vs 93.1%(표)로 다릅니다. 두 서술 모두 원문에 실제로 기재된 내용이며, 어느 쪽이 오기인지 논문이 밝히지 않으므로 그대로 병기합니다.
3. **ResNet 특징벡터 차원(본문 p.5 vs Figure 1)**: 본문(p.5) "1000 features are extracted from the 14th layer of an improved 3D ResNet. A total of 2000 features are finally extracted from both architectures"와 Figure 1의 "FV-4 × 2000" 라벨이 서로 다른 숫자를 제시합니다.
4. **Flatten layer 뉴런 수(Figure 2 내부 산술)**: Figure 2의 Flatten layer 출력이 "12544 neurons"로 표기되어 있으나, 직전 단계 출력 크기(7×7×7, 256채널)를 모두 곱하면 87808이 되어 수치가 맞지 않습니다(7×7×256=12544는 성립). 원문 표기를 수정하지 않고 그대로 인용합니다.
